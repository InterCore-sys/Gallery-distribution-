Key Features

1. Bootstrap System

When you reset the **Arduino** or launch the script on **Pi/Desktop**, you will see a text-based startup sequence loading fake "kernel modules," giving the **authentic feel of a server booting up**.

2. The Terminal (CLI)

It uses a standard prompt. It parses text strings so you can type natural commands.

3. Firewall (FW) Simulation

  • **FW STATUS**: Shows active rules.
  • **FW BLOCK **: Adds an IP address to the mock blocklist.

4. Password Manager (PM)

  • **PM ADD **: Stores credentials.
  • **PM LIST**: Lists stored sites (hiding the actual password for "security").

5. Backup System (Backup)

Visualizes a progress bar and simulates a system dump.

**IMPORTANT NOTE ON DATA:**
* **Arduino:** Data is in RAM and resets on power loss.
* **Pi & Desktop:** Data is stored persistently in a local JSON file.
